Full virtualization
Computing Infrastructure
26
• Concept
• Types, according to the supervisory code execution mechanisms
– Binary translation
– Hardware-assisted
✓Hardware extensions that support hardware-assisted virtualization
Guest operating systems run on VMs without any modification (except for drivers) compared to operating 
systems running on physical platforms.
The supervisory mode code of the operating system is transformed by a translator just before execution, 
so that privileged and IO instructions can be treated appropriately.
The CPU has a special mode for hypervisor execution.  Privileged and IO instructions executed by a guest 
operating system generate exceptions that cause control transfers to the hypervisor, which handles them 
appropriately. 
Intel VT-x y AMD-V

Paravirtualization
Computing Infrastructure
27
This involves modifying the guest operating systems so that they cooperate with the hypervisor for proper 
execution on the VMs.
• Concept
• Mechanism of supervisory mode code execution
• What are the advantages of paravirtualization over hardware-assisted virtualization?
• Scope of use of paravirtualization
• Combined use of hardware-assisted virtualization and paravirtualization
Sections of guest OS code that require executing privileged instructions or performing IO operations are 
replaced by calls to the hypervisor, which handles them appropriately.
Paravirtualization is widely used in the implementation of device drivers. Both Windows and Linux provide 
paravirtualized versions of device drivers for execution on the most common hypervisors.
Most commonly, both technologies are used in combination. Hardware-assisted virtualization for memory 
management and for handling privileged instructions, and paravirtualization in the implementation of 
device drivers.
Paravirtualization offers significantly superior performance in handling IoT devices. Therefore, it is the right 
technology for device driver implementation.

Comparison between platform virtualization and operating 
system virtualization
Computing Infrastructure
32
• Advantages of platform virtualization
– Maximum flexibility in the selection of the operating systems to be run
– Maximum flexibility in the configuration of virtual machines
• Advantages of operating system virtualization
– Lower resource consumption (disk and memory)
– Better performance
Allows the simultaneous execution of virtual machines containing any type of operating system. In other 
words, different operating systems can run simultaneously on the same physical machine.
The ability of virtualization platforms to emulate virtual hardware provides total flexibility in choosing the 
configuration of each virtual machine to be run. 
Containers do not need to maintain their own OS image. They all share a single OS image, which is 
maintained on disk and in memory. However, each virtual machine requires its own OS, which consumes 
the corresponding space, both on disk and in memory. 
Containers do not require managing a complete OS, as they all share a common OS image. This simplifies 
the management of hardware structures for OS management, thus achieving a more efficient 
management of the hardware platform.

• Use

RAID 5: Parity
Computing Infrastructure
61
-Operation: 
-Given a RAID of N disks, for each (N-1) strips of data to be written, a parity strip is
generated.
-The parity strip is calculated by performing the XOR operation on the corresponding
data strips.
-Data strips are distributed sequentially on the disks.
-Parity strips are interleaved between the data strips so that they are evenly
distributed among all disks. 
-If a data strip is lost, (due to a disk failure), the information of that strip can be 
calculated using the rest of data strips of the group and the parity strip. 
-Redundancy: Yes
-Storage efficiency: (N-1)/N 
-Performances in reading operations: (N-1)xP 
-Performances in writing operations: (N/4)xP

This is the ability to make read-only copies of volumes. They are made instantaneously, 
regardless of the size of the source disk. Initially no data blocks are copied in the snapshot. 
Data is only copied when it is modified on the original volume.
This is to create a complete replica of one volume on another. These copies are read and 
write.
This is the ability to set the volumes (identified by their corresponding LUNs) that will be 
visible to each server connected to the network. This facilitates server configuration tasks 
and avoids unnecessary errors.
This is the ability to keep one or several volumes of a storage cabinet permanently replicated in 
another remote cabinet. Replication between each source and remote volume is automatically 
coordinated by the storage cabinets.

Remote replication
20
• Purpose of remote replication
Replicate data generated on a volume by one or more services located at a primary site to another volume 
located at a remote site, so that if a contingency occurs at the primary site, the services can start up and 
continue to operate at the remote site, thanks to the replicated data, until the contingency at the primary 
site is resolved.

iSCSI in front of FC
51
• Advantages
– Same communication technology as that used in data networks
– More personnel trained in IP networks
– Availability of more network management tools
– Lower cost
• Disadvantages
– Significantly higher CPU utilization for traffic management
– TCP/IP protocol generates a significant traffic overhead
– Ethernet switch latency is high
• Conclusions
– Critical or large installations -> FC
– Small and medium installations -> iSCSI
